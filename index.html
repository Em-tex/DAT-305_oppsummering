Module 1: Introduction to Machine Learning

Foundations, Types of Learning, and Python Ecosystem

1.1 What is Machine Learning?

Unlike traditional programming where you hard-code rules, Machine Learning (ML) involves learning patterns from data to make predictions.

Data Input
→
Training Algorithm
→
Model
→

1.2 Types of Learning

Supervised Learning

Training with Labeled Data (Input + Correct Output).

  • Regression: Predicting continuous values (e.g., House Price, Temp).
  • Classification: Predicting categories (e.g., Spam vs Not Spam).
Unsupervised Learning

Training with Unlabeled Data. Finding hidden structures.

  • Clustering: Grouping similar data points.
  • Dimensionality Reduction: Simplifying data (PCA).
Reinforcement Learning

An agent learns to interact with an environment to maximize a Reward.

Module 2: Mathematics for AI

Linear Algebra, Data Representation & Dimensionality Reduction

2.1 Linear Algebra Basics

Vector: A 1D array representing features. Has magnitude and direction.

Matrix: A 2D array. Rows usually represent samples, columns represent features.

Dot Product: $\mathbf{a} \cdot \mathbf{b} = \sum a_i b_i$

Interpretation: If Dot Product is 0, vectors are Orthogonal.

2.2 Data Representation

One-Hot Encoding

Converts categories to binary vectors.

Result: Sparse Vector.

[0, 1, 0, 0]

Embeddings

Converts words/items to dense vectors based on similarity.

Result: Dense Vector.

[0.2, -0.5, 0.9]

2.3 Dimensionality Reduction (PCA)

Principal Component Analysis (PCA) reduces the number of features while keeping the most important information (Variance).

  1. Standardize Data (Mean = 0).
  2. Compute Covariance Matrix.
  3. Calculate Eigenvectors and Eigenvalues.
  4. Select top $k$ components.

Module 3: ML Algorithms

Regression, Classification, Metrics & Clustering

3.1 Algorithms Overview

Linear Regression

Predicts a continuous value $y$ based on input $x$.

Cost Function: MSE (Mean Squared Error). We want to minimize this.

$$MSE = \frac{1}{n} \sum (y_{actual} - y_{pred})^2$$

KNN (K-Nearest Neighbors)

Classifies a new point based on the majority class of its $K$ nearest neighbors.

Lazy Learner: Does not train a model, just stores data.

K too small? Overfitting.

Decision Trees

Splits data into branches.

Goal: Reduce Entropy (Increase Purity).

SVM (Support Vector Machine)

Finds the best hyperplane to separate classes with the maximum Margin.

Kernel Trick: Maps data to higher dimensions to separate non-linear data.

3.2 Evaluation Metrics (Confusion Matrix)

Predicted PositivePredicted Negative
Actual PositiveTrue Positive (TP)False Negative (FN)
Actual NegativeFalse Positive (FP)True Negative (TN)

Accuracy: Overall correctness.

Precision: Quality of positive predictions.

$$TP / (TP + FP)$$

Recall (Sensitivity): Ability to find all positives.

$$TP / (TP + FN)$$

3.3 Clustering (K-Means)

Unsupervised learning to group data into $K$ clusters.

  1. Initialize $K$ centroids randomly.
  2. Assignment: Assign points to nearest centroid.
  3. Update: Move centroid to the mean position of its points.
  4. Repeat until convergence.

Module 4: Neural Networks

Perceptrons, Backpropagation & Training

4.1 The Artificial Neuron (Perceptron)

Modeled after biological neurons. It takes inputs, weighs them, adds a bias, and fires via an activation function.

x
→ w →
$\sum$
→ Activation →
y
$$y = f(\sum (x_i \cdot w_i) + b)$$

4.2 Activation Functions

Essential to introduce Non-Linearity.

  • Sigmoid: Output (0, 1). Problem: Vanishing Gradient.
  • ReLU (Rectified Linear Unit): Output $max(0, z)$. Fast, solves Vanishing Gradient.
  • Softmax: Used in Output layer for multi-class probability distribution.

4.3 Training the Network

Backpropagation: The algorithm to calculate gradients (error contribution of each weight).

Gradient Descent: The optimization method to update weights to minimize loss.

Epoch: One full pass through the dataset.

Batch: A subset of data processed before updating weights.

Module 5: Deep Learning

CNN, RNN, LSTM & Autoencoders

5.1 CNN (Convolutional Neural Networks)

Best for Grid Data (Images).

  • Convolution: Extracts features (edges, textures) using Filters/Kernels.
  • Pooling (Max/Average): Down-samples (reduces size) to prevent overfitting and reduce computation.

5.2 RNN & LSTM (Sequence Data)

RNN (Recurrent Neural Network): Has memory (loops). Good for Time Series, Text.
Problem: Vanishing Gradient (Short-term memory only).

LSTM (Long Short-Term Memory): Solves RNN issues using Gates:

  • Forget Gate: What to discard?
  • Input Gate: What to store?
  • Output Gate: What to pass to next state?

5.3 Autoencoders

Unsupervised learning for Data Compression / Denoising.

Structure: Encoder → Bottleneck → Decoder.

Module 6: NLP

Natural Language Processing & Transformers

6.1 NLP Basics

  • Tokenization: Breaking text into words/sub-words.
  • Stop Words: Removing common words (the, is, a).
  • Stemming/Lemmatization: Reducing words to root (Fishing -> Fish).

6.2 Feature Extraction

TF-IDF: Weighs words by importance. High score = Frequent in document, rare in corpus.

Word Embeddings (Word2Vec): Mapping words to vectors based on semantic meaning. (King - Man + Woman = Queen).

6.3 Transformers

State-of-the-art architecture (BERT, GPT).

Self-Attention: Allows the model to look at the entire sentence at once and weigh relationships between words.

Hugging Face: Popular library/hub for pre-trained Transformer models.

Transfer Learning: Fine-tuning a pre-trained model on your specific task.

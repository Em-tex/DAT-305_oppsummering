1.2 Overfitting & Regularization

Underfitting

Model is too simple. Cannot capture the trend (High Bias).

Balanced

Captures signal, ignores noise.

Overfitting

Model memorizes noise. Fails on test data (High Variance).

Linear Algebra, PCA & Recommender Systems

2.1 Matrix & Vector Basics

  • Square Matrix: A matrix with an equal number of rows and columns ($N \times N$).
  • Dot Product: Measures similarity. If 0, vectors are Orthogonal (Independent).
Orthogonal (0 similarity)

2.2 Dimensionality Reduction (PCA)

Reduces complexity and computational cost by projecting data onto "Principal Components".

1. Standardize
Scale data (Mean=0, Variance=1).
2. Covariance
Calculate relationships.
3. Eigen-Decomposition
Find Eigenvectors (Directions) & Eigenvalues (Importance).
Explained Variance

The amount of information (variance) captured by each Principal Component.

2.3 Recommendation Systems (Matrix Factorization)

How does Netflix recommend movies? By filling in the blanks in a huge matrix of Users vs Movies.

Collaborative Filtering

Predicting interests by finding patterns in user behavior (ratings). Often uses Matrix Factorization.

Matrix Factorization

Decomposing a large matrix (Users x Items) into lower-dimensional matrices.

Algo: SVD (Singular Value Decomposition).

Advantage: Handles Sparse Data well (when users have only rated a few items).

Module 3: Algorithms

Classic ML Models & Evaluation

3.1 Linear Models & Classification

  • Linear Regression: Assumes a linear relationship. Minimizes MSE (Mean Squared Error).
  • Classification: Assigning labels (Cat/Dog).
  • Multi-class Metric: For general classification, Accuracy is common. For imbalanced data, use F1-Score.

3.2 SVM (Support Vector Machines)

Goal: Find the hyperplane with the largest Margin between classes.

The Kernel Trick

Used to handle non-linear data by mapping it to a higher-dimensional space where it becomes linearly separable.

3.3 Confusion Matrix

A table used to describe the performance of a classification model (TP, TN, FP, FN).

MetricUse when...
RecallFinding ALL positives is critical (Cancer, Terror).
PrecisionFalse alarms are expensive (Spam filter).

4.2 Training Process (Gradient Descent)

  1. Forward Pass: Compute prediction.
  2. Loss Function: Measure error (Prediction vs Actual).
  3. Backpropagation: Calculate gradients (direction to change weights).
  4. Update: Adjust weights to minimize error.

Goal: Reach Global Minima (Low Error)

Epoch: One complete pass through the entire training dataset.

Module 5: Deep Learning

CNN, RNN & Autoencoders

5.1 Deep Learning Advantages

Deep Learning = Neural Networks with many layers.

Big Advantage: Automatic Feature Extraction from raw data (no need to manually design features).

5.2 CNN (Images)

  • Conv Layer: Captures spatial features (edges, textures).
  • Pooling Layer: Reduces spatial dimensions (down-sampling). Max Pooling is most common.

5.3 RNN & LSTM (Sequences)

RNN: Good for sequential data (Text, Time Series). Suffer from Vanishing Gradient.

LSTM: Solves this. The Cell State carries long-term information and is updated at each step.

5.4 Autoencoders

Goal: Reduce dimensionality while retaining features (Compression).

Bottleneck: The middle layer with the smallest number of neurons.

Module 6: NLP

Natural Language Processing

6.1 Core Concepts

  • Tokenization: Dividing text into smaller parts (tokens).
  • Lemmatization: Converting words to their root form.
  • TF-IDF: Term Frequency measures frequency of a term in a specific document.
  • Word Embedding: A vector representing a single word (captures semantic meaning).

6.2 Transformers

Innovation: Attention Mechanism (allows model to focus on different parts of input). Enables parallel processing (unlike RNN).

Multi-head Attention: Applies multiple attention mechanisms in parallel to capture different types of relationships.

BERT: Encoder-only. Bi-directional. GPT: Decoder-only.
